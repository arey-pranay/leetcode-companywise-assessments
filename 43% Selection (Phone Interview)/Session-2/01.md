# Intuition

Instead of simulating the entire queue of students, we observe that students only care about the **type of sandwich (0 or 1)**.

So we can simply **count how many students prefer each type**.

Then iterate through the sandwiches stack:

* If there is at least one student who prefers the current sandwich type, decrement that count.
* If no student prefers that sandwich type, the process stops because no student will take it.

The remaining counts represent the students who cannot eat.

---

# Approach

1. Create an array `prefer[2]` to count students who prefer sandwich type `0` and `1`.
2. Traverse the `students` array and update the counts.
3. Iterate through the `sandwiches` array:

   * If `prefer[s] > 0`, decrement the count.
   * Otherwise break since no student wants that sandwich.
4. Return the sum `prefer[0] + prefer[1]`, which represents students unable to eat.

This avoids queue simulation and works in a single pass.

---

# Complexity

* **Time complexity:**
  O(n)
  We iterate once through `students` and once through `sandwiches`.

* **Space complexity:**
  O(1)
  Only a small fixed-size array `prefer` is used.

---

# Code
```java []
class Solution {
    public int countStudents(int[] students, int[] sandwiches) {
        int[] prefer = new int[]{0,0};
        for(int s : students) prefer[s]++;
        for(int s : sandwiches){
            if(prefer[s] > 0) prefer[s]--;
            else break;
        }
        return prefer[0]+prefer[1];
    }
}
```
